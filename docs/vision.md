# Project Vision

## 1. Background
A big insurance company is planning to design a health tracker application for user to have easier access to their health informations, better convenience to track their healthy life style habits like exercises and diets. The product should also allow users to set goals and check their life style history and their progression. They plaaned for an easier integration to the company's database if the user likes the product and decided to use the company's service.

## 2. Project Goals
(List 3–5 high-level goals. Think value, impact, usability.)

- Goal 1: Allow user to store and access their health information.
- Goal 2: Allow user to set life style goals like exercise goals.
- Goal 3: Allow user to review their life style history and provide a visualised progression comparasion.
- Goal 4: Provide structured data for insurance company or doctor to access user records if needed.

## 3. Primary Users & Stakeholders
- User group 1: Normal people
- User group 2: insurance company
- Other stakeholders: doctors

## 4. Success Criteria
(What measurable outcomes indicate that this system is successful?)

- Criterion 1: Users can easily record and navigate through their records.
- Criterion 2: Product will provide an engaging presentation of progression to ensure consistancy and motivation of users.
